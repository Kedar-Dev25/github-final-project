# Smart Clinic Management System - User Stories

---

## Doctor User Stories

### User Story 1: View Daily Appointments
**Title:** Doctor can view all appointments for today  
**Acceptance Criteria:**
- Doctor logs in successfully
- Doctor sees a list of all appointments for the current day
- Each appointment shows patient name, time, and reason  
**Priority:** High  
**Story Points:** 3

### User Story 2: Update Availability
**Title:** Doctor can update available time slots  
**Acceptance Criteria:**
- Doctor logs in successfully
- Doctor selects a date and modifies available time slots
- Changes are reflected immediately in system  
**Priority:** High  
**Story Points:** 3

### User Story 3: View Patient Details
**Title:** Doctor can view patient details  
**Acceptance Criteria:**
- Doctor selects a patient from appointment list
- Doctor sees patient name, contact info, and past appointment history  
**Priority:** Medium  
**Story Points:** 2

### User Story 4: Add Prescription
**Title:** Doctor can add a prescription for a patient  
**Acceptance Criteria:**
- Doctor selects patient
- Doctor enters prescription details
- Prescription is saved and visible to patient  
**Priority:** High  
**Story Points:** 3

### User Story 5: Cancel Appointment
**Title:** Doctor can cancel an appointment  
**Acceptance Criteria:**
- Doctor selects appointment
- Appointment status changes to "Cancelled"
- Patient receives notification  
**Priority:** Medium  
**Story Points:** 2

## Patient User Stories

### User Story 1: Search Doctor
**Title:** Patient can search doctors by specialty  
**Acceptance Criteria:**
- Patient logs in successfully
- Patient enters specialty or name
- List of matching doctors is displayed  
**Priority:** High  
**Story Points:** 3

### User Story 2: Book Appointment
**Title:** Patient can book an appointment  
**Acceptance Criteria:**
- Patient selects doctor and time slot
- Appointment is saved and visible in patient dashboard
- Doctor is notified of new appointment  
**Priority:** High  
**Story Points:** 3

### User Story 3: View Appointments
**Title:** Patient can view all booked appointments  
**Acceptance Criteria:**
- Patient logs in

**Priority:** Medium  
**Story Points:** 2
